## Boundaries

### Always Do
- Run tests before committing
- Use Conventional Commits: `feat(scope): message`, `fix(scope): message`
- Create feature branches: `feat/`, `fix/`, `refactor/`
- Follow existing patterns in the directory you're editing

### Never Do
- Commit secrets or credentials
- Delete or skip failing tests
- Push directly to main/master
